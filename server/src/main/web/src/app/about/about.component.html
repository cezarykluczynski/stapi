The beginning

STAPI, a Star Trek API, was started in October 2016 by Cezary Kluczyński. It was inspired by two projects: Pokéapi and SWAPI, The Star Wars API, both made by Paul Hallett.

Initially, STAPI was just a research project. It aim was to check if building an API using Memory Alpha as a primary source of data is possible. Is it turned out, it was possible.

REST and SOAP

A second goal was to put out two versions of API, SOAP and REST. It been so since the project started. In 2022, SOAP API was deprecated and no longer received model updates, and in 2023 it was removed, due to the fact that nobody was using it.

Tech stack

STAPI was started with Java 8 and Spring Boot 1.5, but has since moved to in Java 17 and Spring Boot 3.x. It's a fairly common tech stack. Contributions to STAPI are most welcome.

Alpha version

In the end of may 2017 an alpha version of STAPI has been put online. It contained 19 models, like locations, books, and astronomical objects. There was over 23000 entities initially available, including almost 6000 characters and more than 5000 performers.

At the end od 2017, the initial development phase ceased, and more than 56000 entities are being served from that point.

Beta version

STAPI was marked as beta version in May 2018. From now on, it entered a stabilization period during which no changes to existing API model are planned. As with any beta version, bugs can happen, and should be reported on GitHub.

At the end of 2022, STAPI received a major model update, with 16 model getting a version 2 of the REST API. It also received a data update, with content from new series and everything else that happened since 2017.
