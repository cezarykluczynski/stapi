This pages describes how to quickly start using STAPI, without caring much for contracts. If you plan on integrating STAPI into a larger codebase, it would be best to use contracts to generate your client, or to use one of the existing clients.

Limits

Currently there are no enforced limits on STAPI usage. Please consider however that this is a pet project and is not supported by a major infrastructure, so be reasonable using it.

API keys

API keys were supported in the past, but support for them was dropped. If your client expects them, just use null or empty string. Non-null API keys will be ignored anyway.

API layout

For people just wanting to quick jump into using STAPI, REST version is the best choice. Every entity in the API has two REST endpoints per version (V1 or V2), one for searching and one for retrieving an entire entity, along with entities related to it. Available endpoints are listed on this page on the bottom, in Available RESTful services section.

Example requests

Listing

Take, for example, the Season entity. To get all seasons, one should call this URL: /api/v1/rest/season/search.

Searching

Similar to listing entities, sending POST request to the same URL, /api/v1/rest/season/search, allows searching for entities using predefined criteria. Criteria that can be used for searching can be found in API documentation.

When searching STAPI, remember to send POST requests as application/x-www-form-urlencoded.

Single entity

Single entities can also be retrieved, using previously obtained UID. UIDs rarely change over time, but when you store them, they could be no good in a few years time. UIDs are generated based on Memory Alpha and Memory Beta unique page IDs, so when wiki pages gets moved, it's ID can change.

After searching for seasons, you will find that season one of The Next Generation happens to have UID SAMA0000001633. So another call to the API renders details about this season, as well as all episodes from this season: /api/v1/rest/season?uid=SAMA0000001633.

Tips

  • You can prepend pretty to any REST request URL to pretty print JSON, for example: /api/v1/rest/season?uid=SAMA0000001633&pretty.
